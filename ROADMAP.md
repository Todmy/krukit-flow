# Roadmap

Krukit is built and used daily. This is where it's headed.

## Near-term

- **Standalone stages.** Make each stage skill useful on its own, not only inside the full `/krukit-flow` run.
- **Fewer moving parts.** Reduce external dependencies so the suite runs leaner and is simpler to install.
- **Reusable primitives.** Pull the building blocks — reality-check, constitution-check, clarification budget, resumable flow-state — out as skills in their own right.

## Direction

- Keep it simple and light: routing that keeps small changes small, gates that require real evidence, lossless resume across stages.
- Portability to other agent runtimes once the core is self-contained.
