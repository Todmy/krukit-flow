# Constitution: krukit
Version: 1.0.0 | Ratified: 2026-07-11

## Principles

1. **Falsifiable edits** — Structural skill changes MUST land with a lint check proven red→green; behavior changes MUST record a falsifiable prediction (bench task or trace pattern) BEFORE implementation. Prompt-code has no other honest test.
2. **Prompt budget** — Any change that grows a SKILL.md MUST justify the recurring token cost in its commit message; compressing or displacing old text is preferred over adding. Every line loads into every session of every user.
3. **Proof or silence** — Public quantitative or comparative claims MUST cite ≥3 bench replications or third-party evidence; qualitative claims only for trace-confirmed mechanisms. One careless number burns the project's credibility.
4. **Resume compatibility** — Format changes to flow-state/templates MUST read existing files without migration (additions OK; renames/removals not). Symlinked skills hit live sessions instantly; a running flow must never break mid-feature.
5. **Layer-0 floor is untouchable** — No edit may weaken the route-independent invariants (snapshot-before-touch, deliverable gate, never-fabricate); they stay lint-pinned. The only benchmark-proven value class; guardrails inside optional stages are not guardrails.
6. **Knobs earn existence** — New routing triggers, gates, or ceremony MUST cite a recurring pattern (≥2 traces in IMPROVE-LOG or bench). Every knob is harness complexity a weaker model must execute.
7. **Rules, not scores** — Adaptivity MUST use categorical, inspectable triggers; numeric complexity scores are banned. Rules are tunable by krukit-improve; a score is neither inspectable nor falsifiable.
8. **Structure pinned by lint** — Duplicated templates and load-bearing sections MUST be covered by lint-consistency; drift is a bug, not a note. The Task:-drift reached all six copies before a lint existed.
9. **Pinned-vendor benchmarking** — A framework version under measurement is never edited mid-grid; changes become a NEW bench condition. Otherwise we destroy our own A/B data.
10. **Git discipline** — Small discrete commits, one logical change each; never push to remote without explicit permission.
